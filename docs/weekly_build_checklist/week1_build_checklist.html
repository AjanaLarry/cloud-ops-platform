Phase 0 — Before you open the console (15 min)
0
Environment verification
~15 min
Verify AWS CLI is connected — run aws sts get-caller-identity and confirm your account ID returns
If it errors, run aws configure and re-enter your credentials
Terminal
Confirm your console region is set to Canada (Central) ca-central-1 — top-right dropdown in the AWS Console
Every resource you create must be in this region — wrong region is the #1 beginner mistake
Console
Set a billing alarm — go to CloudWatch → Alarms → Create alarm → billing → threshold $10 USD
Protects you from unexpected charges during the build. Do this before creating anything else.
Console
Open your local repo in VS Code — cd ~/cloud-ops-platform && code .

Phase 1 — VPC (20 min)
1
Create the VPC
~20 min
Navigate to VPC → Your VPCs → Create VPC
Search bar at top → type "VPC" → click the service
Console
Create VPC with: Name = cloud-ops-platform-vpc-dev · CIDR = 10.0.0.0/16 · Tenancy = Default
Select "VPC only" — not the wizard. IPv6: No IPv6 CIDR block.
Console
Enable DNS settings — Actions → Edit VPC settings → check both: Enable DNS resolution ✅ Enable DNS hostnames ✅ → Save
Without this, EC2 instances won't get public DNS names and RDS won't resolve correctly
Console
Add tags to the VPC: Project=cloud-ops-platform · Environment=dev · ManagedBy=console · Owner=juwon
Tag everything as you create it — retrofitting tags is painful and Terraform will import these later

Phase 2 — Subnets (25 min)
2
Create 4 subnets across 2 AZs
~25 min
VPC → Subnets → Create subnet → select your VPC → create Public Subnet A: Name=cloud-ops-platform-public-subnet-1a · AZ=ca-central-1a · CIDR=10.0.1.0/24
Console
Add new subnet → Public Subnet B: Name=cloud-ops-platform-public-subnet-1b · AZ=ca-central-1b · CIDR=10.0.2.0/24

Phase 4 — Elastic IPs + NAT Gateways (20 min)
4
Allocate Elastic IPs and create NAT Gateways
~20 min
Allocate Elastic IP #1 — VPC → Elastic IPs → Allocate → Tag: Name=cloud-ops-platform-nat-eip-1a
Note the IP address — you will assign it to NAT GW A in the next step
Console
Allocate Elastic IP #2 — Tag: Name=cloud-ops-platform-nat-eip-1b
Two EIPs — one per NAT Gateway, one per AZ
Console
Create NAT Gateway A — VPC → NAT gateways → Create → Name=cloud-ops-platform-nat-1a · Subnet=public-subnet-1a · Connectivity=Public · EIP=nat-eip-1a
NAT Gateway goes in the PUBLIC subnet — this is critical. Status will say "Pending" for 2–3 min.
Console
Create NAT Gateway B — Name=cloud-ops-platform-nat-1b · Subnet=public-subnet-1b · EIP=nat-eip-1b
Console
Wait for both NAT Gateways to show Status = Available before proceeding to route tables
Refresh the page every 60 seconds. Do not create route tables until both show Available.

Phase 5 — Route Tables (25 min)
5
Create and configure route tables
~25 min
Create Public Route Table — VPC → Route tables → Create → Name=cloud-ops-platform-public-rt · VPC=your VPC
Console
Add route to Public RT — select it → Routes tab → Edit routes → Add route → Destination=0.0.0.0/0 · Target=Internet Gateway → your IGW → Save
0.0.0.0/0 means "all traffic not matched by a more specific route goes here" — the default route to the internet
Console
Associate Public RT with both public subnets — Subnet associations tab → Edit → select public-subnet-1a and public-subnet-1b → Save
Console
Create Private Route Table A — Name=cloud-ops-platform-private-rt-1a
Console
Add route to Private RT A — Destination=0.0.0.0/0 · Target=NAT Gateway → cloud-ops-platform-nat-1a → Save
Private subnet A routes outbound traffic through NAT GW A in the same AZ — keeps traffic local to AZ-A
Console
Associate Private RT A with private-app-1a only
Console
Create Private Route Table B — Name=cloud-ops-platform-private-rt-1b → route 0.0.0.0/0 → NAT GW B → associate with private-app-1b
AZ-B has its own private RT pointing to its own NAT GW — this is the per-AZ reliability pattern
Console
